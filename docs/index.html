Setup One script does it all

Run a single PowerShell command and let the setup handle the rest.

Install in PowerShell

One command. Local setup. Done.

PowerShell

powershell -ExecutionPolicy Bypass -Command "Invoke-Expression ((New-Object System.Net.WebClient).DownloadString('https://raw.githubusercontent.com/tucaz/open-dictate/main/scripts/install.ps1'))"

After installation, start it in the background with:

open-dictate start

Configuration

Simple JSON, no control panel maze.

Edit %APPDATA%\open-dictate\config.json

Start with the defaults, then change only what you need.

The app keeps configuration in one plain JSON file under your Windows profile. No control panel, no account, no hidden sync layer.

Right Ctrl by default base.en for English Privacy mode on Language switchable

Default config example

{
  "hotkey": { "keyCode": 163, "modifiers": [] },
  "modelSize": "base.en",
  "language": "en",
  "spokenPunctuation": false,
  "maxRecordings": 0
}

modelSize

Default is base.en, which is English-only. Available model values are tiny.en, tiny, base.en, base, small.en, small, medium.en, medium, and large.

language

Default is en. Set it to a Whisper language code such as pt, es, fr, de, or ja.

spokenPunctuation

Turn it on if you want phrases like "comma" and "period" converted into punctuation marks.

maxRecordings

Keep it at 0 for privacy-first mode, or raise it to preserve recordings for re-transcription.

Using other languages

The default installer grabs base.en, which is English-only. For any other language, switch to a multilingual model such as base or small, then set the matching Whisper language code in the config.

Typical examples:

  • pt for Portuguese
  • es for Spanish
  • fr for French
  • de for German
  • ja for Japanese

Example flow: if you want Brazilian Portuguese, download base or small, set that as the model, then use "language": "pt". 

open-dictate download-model base
open-dictate set-model base

# then edit %APPDATA%\open-dictate\config.json
{
  "hotkey": { "keyCode": 163, "modifiers": [] },
  "modelSize": "base",
  "language": "pt"
}

For better recognition quality, use small instead of base. The pattern is the same for other languages: keep the model multilingual and swap the language code.

FAQ

What people usually need to know before trying it.

Is it free?

Yes. It is free, open source, and a Windows port of the original open-wispr app for macOS.

Does my audio leave the machine?

No. The intended workflow is local transcription using Whisper on your own machine.

Which Windows versions are supported?

Windows 10 and Windows 11.

What hardware should I start with?

Start with the default base.en model for English, or base for multilingual use. OpenAI lists approximate Whisper memory needs at about 1 GB for tiny/base, 2 GB for small, 5 GB for medium, and 10 GB for large. In practice, that means base.en or base is the sensible choice for most Windows laptops and desktops, small is for stronger machines that want better accuracy, and medium/large are usually overkill for quick dictation.

Will it work in the app I already use?

That is the goal. Dictation is meant to land wherever your active text cursor currently is.

Can I change the hotkey or language?

Yes. Use practical hotkeys like Right Ctrl, Insert, or Ctrl+Space, and edit %APPDATA%\open-dictate\config.json for language changes.

How do I use Brazilian Portuguese?

See the Configuration section above. The same multilingual setup applies there, with Brazilian Portuguese as one example.

I need the macOS version instead.

Use the original project: open-wispr.com or the human37/open-wispr repository.
